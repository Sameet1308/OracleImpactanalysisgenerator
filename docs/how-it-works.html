Connect to Oracle Ecosystem

Three ways to get Oracle artifacts into the system

Path A: Oracle Database Connection

Connect directly to Oracle DB to auto-discover artifacts from DBA_OBJECTS and DBA_SOURCE system views. The system queries for tables, views, procedures, functions, packages, triggers, and sequences.

Demo mode: Loads 5 pre-built sample artifacts simulating a real Oracle HR/Payroll schema with 25 objects and 34 cross-artifact dependencies.

Path B: Upload Oracle Artifacts

Drag and drop actual Oracle ERP files for analysis:

  • SQL / PL-SQL — DDL scripts with CREATE TABLE, VIEW, PROCEDURE, FUNCTION, PACKAGE, TRIGGER
  • OIC XML — Oracle Integration Cloud flow definitions with connections, steps, and queries
  • BIP XML — BI Publisher report definitions with data models and embedded SQL
  • Groovy — HCM/ERP Cloud scripts with executeQuery, executeFunction, executeProcedure calls

Graph Engine — Blast Radius Computation

Edge direction: A → B means "A depends on B". Impact analysis walks predecessors (reverse direction) to find everything that would break if B changes.

compute_impact("EMPLOYEES") Direct: HR_EMPLOYEE_SUMMARY, PAYROLL_VIEW, GET_EMPLOYEE_SALARY, CALC_ANNUAL_BONUS, HR_PACKAGE, PAYROLL_TRIGGER (6 objects) Indirect: COMPENSATION, HR_SYNC, MONTHLY_PAYROLL_REPORT (3 objects via chains) Total blast radius: 9 artifacts affected

Risk Score Formula (0-100)

risk_score = (direct_weight * 0.52) ← 52% weight on direct dependencies + (indirect_weight * 0.18) ← 18% weight on transitive chain + (type_criticality * 0.30) ← 30% weight on object type Type criticality: TABLE=10, PACKAGE=9, PROCEDURE=8, OIC_FLOW=8, VIEW=7, BIP_REPORT=7, GROOVY=6, TRIGGER=6, SEQUENCE=4

80-100: CRITICAL 60-79: HIGH 40-59: MEDIUM 0-39: LOW

RAG Knowledge Base — 2-Stage Retrieval

Stage 1: Exact metadata match — Find code chunks belonging to the target object (score: 1.0)

Stage 2: Semantic search — Vector similarity on impacted object names + source code (top-k=5)

Result: The most relevant code snippets are injected into the AI prompt for context-aware analysis.

Frequently Asked Questions

Where does the data come from?
From real Oracle artifact files (.sql, .xml, .groovy) that are uploaded by the user or loaded from the sample dataset. The system parses these files using regex and AST extraction to discover objects and their cross-artifact dependencies. Nothing is hardcoded.
Can it connect to a live Oracle database?
The architecture supports it (querying DBA_OBJECTS and DBA_SOURCE), and the UI has a connection form for Oracle DB. In the current demo, "connecting" loads the sample dataset which simulates a real Oracle HR/Payroll schema. Full live DB connectivity is on the roadmap.
How is the risk score calculated?
52% from direct dependency count, 18% from indirect (transitive) dependency count, and 30% from object type criticality (TABLEs score highest at 10/10, SEQUENCEs lowest at 4/10). EMPLOYEES gets 82 (CRITICAL) because it has 6 direct dependents and is a TABLE.
What makes this different from Oracle DBA_DEPENDENCIES?
Oracle's built-in DBA_DEPENDENCIES only tracks PL/SQL dependencies within the database. It completely misses OIC integration flows, BI Publisher reports, and Groovy/HCM scripts. This system discovers ALL cross-artifact dependencies across the entire Oracle ecosystem.
How does the AI generate recommendations?
The system builds a rich prompt containing: impact analysis data + actual source code retrieved via RAG (ChromaDB + sentence-transformers). This prompt is sent to BlueVerse Foundry's pre-trained Oracle ERP agent (AI_Elite_Ora1), which generates Oracle-specific recommendations with real ORA error codes.
What if the BlueVerse token expires?
The UI shows token status (green/yellow/red) and has a paste input to refresh the JWT token at runtime without restarting the server. The system also falls back to a rule-based mock engine that generates deterministic analysis for demo purposes.
