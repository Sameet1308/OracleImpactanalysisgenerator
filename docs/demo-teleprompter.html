Problem & Solution Overview

40 sec · deck slides 2-3
On the problem side, DBA_DEPENDENCIES tracks PL/SQL object references inside the database only. It has no visibility into OIC flow XML, BIP data-model XML, or Groovy scripts — artifacts that live outside the database and account for a significant portion of production breakage in modern Oracle deployments.

Our solution is a five-step pipeline. Ingest the artifacts. Parse every object and dependency using four dedicated parsers. Build a unified dependency graph. Score the risk. And finally, generate AI-grounded root cause, fix recommendations, testing steps, and a rollback plan. Let me walk you through the product.

AI-Generated Recommendations

65 sec · governance + grounding
close column modal → click CRITICAL risk pill → AI modal opens
For AI-generated recommendations, the tool routes requests through BlueVerse Foundry, LTM's sanctioned AI platform. The agent in use is AI_Elite_Ora1, configured for Oracle context.

Grounding is handled by a local RAG pipeline: ChromaDB with sentence-transformers runs inside the application container, so source code never leaves the process. Before the prompt is sent to BlueVerse, a PII filter strips credentials, connection strings, JWTs, and email addresses via regex. Prompts are also capped at twelve kilobytes.

The agent returns four structured sections: root cause analysis, five fix recommendations citing real ORA error codes, a testing checklist, and a five-step rollback plan. Each response is scanned for hallucinated ORA codes against a curated whitelist.

Per LTM data-governance policy, BlueVerse is the sole LLM provider — there are no calls to Claude, OpenAI, Gemini, OCI, or AWS Bedrock. If BlueVerse is unreachable, the system falls back to a deterministic local engine.

A PDF report can be generated in one click for attachment to a change-advisory-board ticket.

5:55

Architecture

45 sec · deck slides Arch + AWS
Alt-Tab to pitch deck → Prototype Architecture slide
The prototype architecture: a FastAPI backend exposing thirteen REST endpoints, a NetworkX directed graph for dependency tracking, four purpose-built parsers, and ChromaDB with MiniLM-L6-v2 for local RAG. BlueVerse Foundry is the sole external dependency; all other components run in-process.
flip to Production on AWS slide
The production architecture on AWS: ECS Fargate behind an Application Load Balancer, Aurora Serverless v2 with pgvector consolidating both the dependency graph and the vector index in a single database, S3 for artifact storage, CloudFront for static delivery, Cognito for identity federation, and CloudWatch plus X-Ray for observability.

Estimated steady-state cost: approximately two hundred and thirty dollars per month single-tenant, or fifty to eighty dollars per tenant at multi-tenant scale. AWS Bedrock is intentionally excluded — the BlueVerse-only policy applies in production as well.
